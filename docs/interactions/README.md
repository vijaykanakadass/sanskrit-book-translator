# Interactions

Chronological record of every task performed on this project. Each numbered folder represents one interaction — a request from the user, through planning and implementation, to outcome.

## Structure

Each folder contains up to five files:

- `request.md` — what the user asked for
- `plan.md` — the approach designed before implementation
- `decisions.md` — choices made during the interaction
- `changes.md` — files created, modified, or deleted
- `outcome.md` — what was actually built

Read folders `001/` through `NNN/` in order to get the full project history.

## Convention

- Folders are numbered sequentially: `NNN-short-description/`
- Not every folder needs all five files — small tasks may only have request + changes + outcome
- All content is kept in full detail, never condensed
